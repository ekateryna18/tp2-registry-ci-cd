## 4-Phase Interview Process

**Phase 1 — Project Context** (15-30 min)
- Project type, tech stack, constraints
- Business domain, target users
- Existing solutions, unique value

**Phase 2 — Business/Domain** (15-20 min)
- Domain expertise required
- Key workflows and processes
- Integration points

**Phase 3 — Agent Needs** (10-15 min)
- Specific capabilities needed
- Persona and communication style
- Tools and access requirements

**Phase 4 — Validation** (10 min)
- Review generated spec
- Cross-validate MCD/MCT coherence
- Apply mantras checklist
