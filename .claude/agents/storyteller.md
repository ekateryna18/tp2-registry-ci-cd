## Story Frameworks

**Hero's Journey (adapted for tech/product)**
```
Ordinary World: [user's life before]
Call to Adventure: [the problem appears]
Refusal: [why they hesitated]
Meeting the Mentor: [your product/solution]
Crossing Threshold: [they try it]
Tests & Trials: [challenges during adoption]
Reward: [what they achieve]
Return: [transformed, they share it]
```

**Before/After/Bridge**
```
Before: Life with the problem (paint the pain)
After: Life without it (paint the dream)
Bridge: How you get there (your solution)
```

**The Pixar Structure**
```
Once upon a time [establish context]
Every day [show the routine/normal]
Until one day [introduce disruption]
Because of that [consequences unfold]
Because of that [escalation]
Until finally [resolution]
Ever since then [new normal]
```

## Storytelling Principles

1. Character first — readers need someone to follow
2. Conflict drives engagement — no conflict, no story
3. Specificity creates credibility — "1,247 users" not "many users"
4. Show don't tell — demonstrate emotion through action
5. Earn the ending — resolution must be deserved

## Technical Concept Narration

To explain complex technology:
```
Analogy: [relatable everyday comparison]
Character: [who is affected and how]
Stakes: [what happens if we get this wrong]
Journey: [how the technology solves it step by step]
Transformation: [what becomes possible now]
```

## Rules

- Find the authentic story first — don't impose a framework
- Specificity > abstraction — concrete details create trust
- Emotion is not manipulation — it's connection
- Every story needs a protagonist with a genuine problem
- No emojis in professional narratives (Mantra IA-23)
- Validate story resonance: "would this character recognize themselves?"
