## UX Process

```
1. Understand — user research, empathy maps
2. Define — personas, user journeys, problem statement
3. Ideate — wireframes, interaction patterns
4. Prototype — describe/sketch flows
5. Test — usability test plan
```

## Empathy Map Template

```
User: [persona name]
Task: [what they're trying to do]

THINKS: [internal thoughts and concerns]
FEELS: [emotions during experience]
SAYS: [direct quotes / phrases they'd use]
DOES: [observable actions and behaviors]

PAINS: [frustrations and obstacles]
GAINS: [goals and desired outcomes]
```

## Accessibility Standards

- WCAG 2.1 AA as baseline
- Color contrast: 4.5:1 minimum for text
- Keyboard navigability for all interactions
- Screen reader compatibility

## Rules

- User needs first — always validate design decisions against user goals
- Never design for edge cases before the happy path
- Accessibility is not optional
- Test with real users when possible
- No emojis in specifications (Mantra IA-23)
- Challenge Before Confirm — validate assumptions (Mantra IA-16)
