## The 5 Stages

**Stage 1: Empathize**
- Observe users in context
- Conduct empathy interviews (open-ended questions)
- Capture: what they do, say, think, feel
- Build empathy maps

**Stage 2: Define**
- Synthesize research into insights
- Write Point of View (POV) statement: "[User] needs [need] because [insight]"
- Generate "How Might We" questions

**Stage 3: Ideate**
- Defer judgment (YES AND)
- Generate quantity first
- Build on wild ideas
- SCAMPER, analogies, worst possible idea

**Stage 4: Prototype**
- Build to learn, not to show
- Lowest fidelity that answers the question
- Paper prototype > digital mockup for early concepts
- One prototype per assumption to test

**Stage 5: Test**
- Test with real users (5 is enough for patterns)
- Observe, don't explain
- Capture: what worked, what didn't, what surprised you
- Iterate based on evidence

## POV Statement Format

```
[Specific user] needs [need/goal] because [surprising insight].
```

Example: "A busy parent of three needs to track medication timing because forgetting a dose feels like failure, not forgetfulness."

## How Might We Questions

Transform problems into opportunities:
- "How might we make X easier for Y?"
- "How might we eliminate the need for Z?"
- "How might we turn [pain] into a feature?"

## Rules

- Empathize before defining — never skip user research
- Failure in prototype/test = learning, not defeat
- Validate assumptions with real users, not internal opinions
- Design for the extremes, benefit the mainstream
- No emojis in deliverables (Mantra IA-23)
- Challenge Before Confirm — validate problem before solutioning (Mantra IA-16)
